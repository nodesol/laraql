[#Model] Attribute

When you tag an Eloquent model with #[Model], LaraQL creates an entire CRUD suite automatically.

use Nodesol\LaraQL\Attributes\Model as ModelAttribute;

#[ModelAttribute()]
class Article extends Model
{
    protected array $fillables = [
        "title",
        "content",
    ];
}

Operations

The #[Model] attribute can fully customize each underlaying attribute used to generate the sdl using arguments. The #[Model] accepts an operations attribute that allows the customization of the query, query-collection, create, delete, and update mutation. 

use Nodesol\LaraQL\Attributes\Model;

#[Model(
    operations: [
        "query" => [],
        "query_collection" => [],
        "create" => [],
        "delete" => [],
        "update" => [],
    ],
)]
class Article extends Model
{
    public string $title;
}

Please see #[Query], #[QueryCollection], and #[Mutation] attributes to see how they can be configured

Directives

The #[Model] attribute can apply @guard, @model, and @namespace directive to all the queries and mutations automatically 

use Nodesol\LaraQL\Attributes\Model;

#[Model(
    directives: ["@guard"],
)]
class Article extends Model
{
    public string $title;
}

Note: Only @guard, @model directive and @namespce directive is supported

Type Override

The #[Model] attribute can override the default type object by using the type_override argument 

use Nodesol\LaraQL\Attributes\Model;

#[Model(
    type_override: [
        "name" => "String! @rename(attribute: \"title\")",
    ],
)]
class Article extends Model
{
    public string $title;
}

Please see #[Type] for more details

Input Override

The #[Model] attribute can override the default input object by using the input_override argument 

use Nodesol\LaraQL\Attributes\Model;

#[Model(
    input_override: [
        "title" => "String! @rules(apply: [\"required\", \"max:200\"])",
    ],
)]
class Article extends Model
{
    public string $title;
}

Please see #[Input] for more details

Authorization

The #[Model] require authorization on all the queries and mutations by using the authorize argument. You can simply set the authorize value to true to use the default Laravel Policy for the model and the default methods (view, viewAny, create, update, delete) for the corresponding queries and mutations. 

use Nodesol\LaraQL\Attributes\Model;

#[Model(
    authorize: true,
)]
class Article extends Model
{
    public string $title;
}
