[#Input] Attribute

When you add #[Input] attribute to a model or a class, LaraQL creates a GraphQL input type automatically using the class's properties or Model's fillables as input fields. You can use any built-in php types or any class that has a corresponding GraphQL Type as the type of your input field. You can also make your constructor arguments optional for the input fields to be optional in the GraphQL Type

By default Lighhouse GraphQL package looks for inputs in the default namespace App\GraphQL\Inputs

namespace App\GraphQL\Inputs;
                    
use Nodesol\LaraQL\Attributes\Input;

#[Input()]
class Article
{
    public function __construct(
        public int $id,
        public string $name,
    ){}
}

Name

The #[Input] attribute can fully customize each underlaying attribute used to generate the sdl using arguments. The #[Input] attribute accepts a name attribute that allows you to override the name of the input type. By default the class name is used and the word Input is appended. Here is how we create and ArticleInput in GraphQL 

namespace App\GraphQL\Inputs;
                    
use Nodesol\LaraQL\Attributes\Input;

#[Input(
    name: "ArticleInput",
)]
class Article
{

}

Note: The names must be unique. You can not have a type and an input with the same name.

Inputs

The #[Input] attribute can be given a list of inputs or field names using the inputs argument. The inputs argument expects an array of inputs with the keys being the field names and values being the type of field along with any directives that need to be added to the field 

namespace App\GraphQL\Inputs;
                    
use Nodesol\LaraQL\Attributes\Input;

#[Input(
    inputs: [
        'title' => 'String! @rules(apply: [\"required\", \"max:200\"])',
    ],
)]
class Article
{

}
Supported Directives
Directive Description
@convertEmptyStringsToNull Use this when there is no meaningful distinction between an empty string and null.
@deprecated Specifies a field to be deprecated
@drop This is useful when you want to deprecate a field, but avoid breaking changes for clients that still pass the value.
@globalId A unique base64-decoded string that globally identifies the object
@hash The most common use case for this is when dealing with passwords
@method Useful for models or other classes that have getters. It will call the given method with the client given arguements
@rename This can often be useful to ensure consistent naming of your schema without having to change the underlying models.
@rules Validate an input using Laravel validation.
@rulesForArray Apply validation rules to each item in an array using Laravel validation.
@spread Merge the fields of a nested input object into the arguments of its parent when processing the field arguments given by a client.
@validator Provide validation rules through a PHP class

Inputs Override

The #[Input] attribute can override the default fields by using the inputs_override argument to specify a different GraphQL Type or apply directives. It can also be used to add additional fields. 

namespace App\GraphQL\Inputs;
                    
use Nodesol\LaraQL\Attributes\Input;

#[Input(
    name: "ArticleWithCommentsInput",
    inputs_override: [
        "comments" => "[CommentInput]",
    ],
)]
class Article
{
    public function __construct(
        public int $id,
        public string $name,
    ){}
}
